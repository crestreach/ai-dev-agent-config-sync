# cyncia 1.1.0

Feature release on top of [v1.0.1](v1.0.1.md). Adds first-class Codex
support across sync targets, expands `cyncia.conf`, normalizes embedded rule
headings in aggregate guideline outputs, and refreshes docs/tests around the
new six-tool matrix.

## Highlights

- **Codex support.** `sync-all` now supports `codex` alongside Cursor,
  Claude Code, GitHub Copilot, VS Code, and JetBrains Junie:
  - `agents/<name>.md` -> `.codex/agents/<name>.toml`
  - `skills/<name>/` -> `.agents/skills/<name>/`
  - `mcp-servers/<name>.json` -> `.codex/config.toml` `mcp_servers` tables
  - `AGENTS.md` copied to the output root
  - Markdown `rules/*.md` merged into root `AGENTS.override.md` when
    `codex-rules-mode: agents-override`
- **Codex MCP translation.** Codex MCP sync writes TOML config while preserving
  unrelated `.codex/config.toml` settings. Generated servers are added or
  updated without `--clean`; with `--clean`, generated `mcp_servers` are
  replaced by the selected source servers.
- **Codex rule behavior aligned with Codex docs.** Cyncia does not generate
  `.codex/rules` from Markdown rules because Codex native `.rules` files are
  Starlark command-approval policy. Markdown guidance goes to
  `AGENTS.override.md`, which Codex prefers over `AGENTS.md` in the same
  directory.
- **Hyphenated `cyncia.conf` keys.** Config keys now use hyphens consistently:
  - `claude-rules-mode`
  - `codex-rules-mode`
  - `codex-sync-mcp`
  - `default-tools`
- **Embedded rule heading normalization.** When rule bodies are merged into
  `CLAUDE.md`, `.junie/AGENTS.md`, or `AGENTS.override.md`, Cyncia shifts ATX
  headings so the rule body's highest heading starts below the generated
  per-rule wrapper. Standalone rule files preserve authored heading levels.

## Configuration changes

`cyncia.conf` now supports:

```yaml
claude-rules-mode: claude-md
codex-rules-mode: agents-override
codex-sync-mcp: true
default-tools: cursor,claude,copilot,vscode,junie,codex
```

The installer schema creates these defaults when `cyncia.conf` is missing,
prompts to add missing properties on upgrade, and prompts before removing
unsupported properties.

## Tests

- Bats coverage expanded for Codex sync, Codex MCP config merging, Codex agent

- Pester coverage expanded for the PowerShell Codex/MCP paths and aggregate
  heading normalization.

Total: 86 bats tests + 43 Pester tests.

## Upgrading from 1.0.1

Re-run the installer to refresh `.cyncia/` and update `cyncia.conf`:

```bash
curl -fsSL https://raw.githubusercontent.com/crestreach/cyncia/main/install/install.sh | bash
```

If you already opted into Claude `rule-files` mode in v1.0.1 using the old
underscore key, rename it from:

```yaml
claude_rules_mode: rule-files
```

to:

```yaml
claude-rules-mode: rule-files
```

Then re-run `sync-all`.
